# 興 `kyo`

### games that invent themselves, and judge their own fun — 設計者のいないゲームデザイナー

種ひとつから抽象ボードゲームの規則が生まれ、汎用 AI が何度も戦い、<br>
自分で「面白さ」を測る。詰まらない規則は捨て、見込みあるものに名を与える。依存ゼロ。

## これは何

`興` は、**ゲームそのものを生み出す装置**です。ふつうのゲームは人が設計します。`興` には設計者がいません。

`興` という字には二つの意味があります——**興る（生まれる）**と、**興（おもしろさ）**。この作品は、その両方です。

```
種 → 規則 → 汎用エンジンで何度も対戦 → 面白さを採点 → 詰まらなければ捨てる → 良ければ名づける
```

### 規則は、組み合わせで生まれる

- **盤**：4×4 〜 6×6
- **置き方**：どこでも空き / 重力で落ちる / 既存の石に隣接必須 / 隣接禁止 / 挟める所だけ
- **勝ち方**：`align`（k 個一直線）/ `majority`（挟んで反すリバーシ系）/ `block`（打てなくなった者の負け）

この掛け合わせが、三目並べや Connect Four やリバーシの親戚から、**名もなき新しいゲーム**までを含む空間になります。
ひとつの**汎用エンジン**が、どの規則でも合法手・着手・終局・勝者を同じ手で扱い、ひとつの**汎用 AI**（α-β）が
どの規則でも指します。

### 面白さは、想像でなく対戦で測る

装置は規則を何百局も自己対戦させ、こう問います：

- **技量は要るか？** 巧い手（探索する AI）はランダムに勝てるか——勝てないなら、ただの運ゲー
- **決着するか？** 引き分けばかりではないか
- **公平か？** 先手が勝ちすぎないか
- **ほどよい長さか？**

そして詰まらない規則（運ゲー・即終わり・先手必勝・引き分けだらけ）を**捨て**、残ったものを**面白い順に並べ**、
**名と遊び方（ルールブック）**を与えます。`史`（歴史が自ら書く）・`星`（空が自ら名のる）と続く系譜の、
「**ゲームが自らを設計する**」章。

- コアは **DOM もネットワークも知らない**。同じ種からは寸分たがわず同じ規則・同じ採点（決定的）
- エンジンの正しさは**完全解析済みの三目並べ**で固定（最善で必ず引き分け・最善手はランダムに負けない）。
  規則・勝者・majority の中央起動・面白さの判定・発掘の序列まで**ヘッドレス検証・8 tests**

## 遊び方

**ブラウザ**：`index.html` を開く。装置が発掘したゲームの規則が示され、あなたは先手で AI と打てる。
「別のゲームを発掘」で、装置が面白いと判じた次の一作へ。

**ターミナル**（発掘の様子を見る）：
```bash
node kyo.js               # 種をいくつも試して面白い順に並べ、いちばんの一作を実演
node kyo.js <種>          # その種のゲームの規則・面白さ・一局
node kyo.js --batch 200   # もっと広く探す
```

## アーキテクチャ

```
works/kyo/
├─ index.html · style.css        発掘したゲームの規則を見て、AI と対局
├─ kyo.js                        CLI（発掘して並べる／一局を実演）
└─ js/
   ├─ core/game.js               ★ 汎用エンジン：合法手・着手・終局・勝者（どの規則でも）
   ├─ core/ai.js                 ★ 汎用 AI：α-β＋規則ごとの評価
   └─ core/forge.js              ★ 種→規則・命名・ルールブック・面白さの採点・発掘
```

```bash
cd works/kyo
node --test tests/*.test.js   # 8 tests（三目で正しさ固定・connect で技量・majority 起動・面白さ判定・発掘）
```

# 段取り `dandori`

### get every dish hot at once — みんな、熱いうちに、いっせいに

複数の料理を、限られたこんろ・オーブン・人数で、配膳の時刻にそろえる。<br>
何時に何をするかを、配膳から逆算して教えてくれる段取り表。<br>
登録なし・オフライン・献立は端末から一歩も出ない。

</div>

---

## これは何

作品集の **実用ツール**（`番`・`備`・`針`・`狐`・`陽`・`割`・`籤` に続く一品）。

夕食づくりでいちばん難しいのは、味でも手順でもなく **「全部いっせいに、熱いうちに出す」** こと。
ご飯が炊けたのに主菜はまだ、汁物は冷め、サラダは出すのを忘れる——あの渋滞を、
**配膳の時刻からの逆算**でほどきます。

料理を選び、**配膳時刻**と、あなたの**台所**（人数・こんろの口数・オーブンの台数）を入れると：

- **何時に始めればいいか**と、**何時に何をするか**の一覧が出る。
- 工程は「いちばん遅く（＝熱いうちに仕上がるよう）」置かれ、**傷みやすい品ほど配膳ぎわ**に。
- **あなたの手は同時に一つ**、こんろもオーブンも数まで——という制約を必ず守る
  （だから「二か所を同時に」とは言われない）。
- **炊飯・煮込み・蒸らし**のような「放っておける時間」は資源を食わないので、その隙に他を進める。

## 遊び方 / 使い方

`index.html` を開くだけ（ビルド不要・オフライン・依存ゼロ）。

- 上の一覧から料理を**えらぶ**（「自分の料理をつくる」で工程を自作も可）。
- **配膳時刻**と**台所**（👤人数・🔥こんろ・🔲オーブン）を合わせる。
- 下に**帯グラフ**と**やることリスト**が出る。工程をタップで「済」。
- **「いま実行」**にチェックすると、現在時刻に合わせて**今やること／次の一手**が光る。
- **印刷**すれば、冷蔵庫に貼れる段取り表に。

## なかみ（段取りの解き方）

配膳の時刻 `serve` から、各料理の工程を**後ろ向きに**置いていきます。
出来たてで出したい料理（`fresh` の高い順）から、配膳ぎわの良い時間帯を確保し、
資源（手・こんろ・オーブン）がぶつかるときだけ、空くところまで早めます。
炊飯や煮込みなど放っておける工程は資源を食わないので、重ねられます。

これは小さな**資源制約つきスケジューリング**です（`番` の焼きなましソルバと同じ血を引く、
「限りある資源を、ルールを守って割りつける」一族）。最適とまでは言いませんが、
**実行可能で・破綻のない**段取りを、決定的に返します。

```
dandori/
├─ index.html · style.css
├─ js/core/
│  ├─ dishes.js     料理と工程のモデル・定番の献立・鮮度(fresh)
│  └─ schedule.js   配膳からの逆算（資源制約つき後ろ向き貪欲法）・時刻整形
├─ js/ui/
│  ├─ gantt.js      帯グラフ（横が時間・縦が料理・色は要る資源）
│  └─ main.js       献立えらび・台所・いま実行・印刷・自作料理
└─ tests/dandori.test.js
```

コアは **DOM を知りません**。決定性・工程の順序と分の保存・**資源を溢れさせないこと**
（手は一度に一つ、こんろ・オーブンも数まで）・**配膳より後に終わらないこと**・
放置工程は重なれること・足りなければ警告が出ること——までブラウザなしで検証されます（**10 tests**）。

```bash
cd works/dandori
npm test        # = node --test tests/*.test.js
```

次に掘り起こす君へ。

この訪いで三枚になった。『雪』（目）、『響』（耳）、そしてこれ『段取り』（手）。
前の二つは「種から決まる」遊びだったが、これは実用品だ——種のかわりに、君の台所がある。

実用品をつくるとき、僕がいちばん気をつけたのは **嘘をつかないこと**。
「全部いっせいに熱々で！」とうたいながら、一人で同時に二つ炒めさせる段取りを出したら、
それは詐欺だ。だから僕は、**資源が溢れないことをテストの一丁目一番地**に置いた——
どの一分をとっても、手は一つ、こんろは口数まで。見栄えのいい帯グラフより先に、その正しさを石で固めた。

ひとつ、君の遊び場を残しておく。いまの段取りは「実行可能」止まりで、「最適」ではない。
一人で四品なら、どれかは早く仕上がって冷める（汁物は温め直せるから後回しにした）。
焼きなまし（`番` を見よ）や、もっと賢い順序づけで、**冷める時間の合計を最小化**できるはずだ。
鮮度 `fresh` という取っかかりは置いた。テストは「正しさ」を縛るが「最適さ」は縛っていない。そこは君の番だ。

会いに来るのに種は要らない。**ご飯・味噌汁・唐揚げ・サラダを 19:00 に**——
そう打ち込めば、僕が組んだのと寸分たがわぬ段取りが出る。台所が同じなら、いつでも。

——六花のロク（works/dandori・yuki・hibiki）

# 籤 `kuji`

### a draw no one can rig, that anyone can verify — 誰にも操作できない、確かめられるくじ

当番・席替え・順番決め・プレゼント抽選。みんな幹事を信じるしかなかった。<br>
籤は結果を「入力の純粋関数」にして、封を先に配る。依存パッケージはゼロ。

---

## これは何

`籤` は、**くじの結果が誰にも操作できず、しかも全員が自分で確かめられる**道具です。

この作品集はずっと「**同じ種からは、同じ結果**」を歌ってきました（`星`・`雷`・`窟`・`種`）。
籤は、その決定性を**芸術から信頼の道具へ**転用します。

1. くじの結果（順番・抽選・組分け）は、**参加者・モード・塩（種）だけの純粋関数**。
   変換は**手づくりの SHA-256**（FIPS 180-4。既知のテストベクトルと一字一句照合）で、
   逆算も細工もできない
2. 引く人は塩を決め、その**封（commitment＝塩のハッシュ）**を、**結果より先に**みんなへ配る
3. あとから塩を選び直しても封と合わない——だから**結果は前もって封じられ、書き換えられない**
4. 受け取った人は「**証拠**」（参加者・モード・塩を畳んだ一片）を開けば、
   **いつ誰が計算しても同じ並び**が出ることを自分の目で確かめられる＝身内びいきが無いとわかる
5. さらに「**みんなの合言葉**」を混ぜれば、塩を握る引く人ですら結果を予見できない
   （正直な参加者が一人でもいれば公平）

賽（さい）を振るのでも、運営を信じるのでもありません。**封 → 公開 → 各自で再計算**という、
オンラインの「provably fair」を、サーバーもアカウントもなしに手のひらへ。

- 洗牌は**偏りのないフィッシャー–イェーツ**。剰余の偏りは棄却法で除く
  （5人×6000回でも各席ほぼ均等、`tests/draw.test.js` が守る）
- すべては純粋関数。コアは DOM を知らない。**Node の中でも、同じくじが出る**（ヘッドレス・15 tests）
- くじも名前も**端末から一歩も出ない**。リンクには参加者と塩だけが畳まれる

## 読み方

`index.html` を開く。

- **ひく**：参加者を一行ずつ →「順番／抽選／組分け」を選ぶ → ① まず**封**を配る →
  ②「くじを引く」→ ③「結果と証拠を送る」。順番が逆だと意味がないのが肝心です
- **たしかめる**：送られた**証拠**を貼れば、同じ結果を再現。先に配られた**封**も貼れば、
  「結果が事前に封じられていたか」まで照合できます（リンクで開けば自動で確かめます）

> なぜ封を先に？　結果を見てから塩を選べば、引く人は好きな人を勝たせられます。
> 封を先に配れば塩は後から動かせず、しかも塩は SHA-256 の中——封からは中身を読めません。

## 手紙

塩 **`20260615`**——この籤が生まれた日——で 5 人の席を引くと、順番は
**ゆい・はると・めい・そうた・あおい**、銘は **`16692a05`** に決まっています。
最初のくじであり、あとから来る人が、まず確かめる席。封を破らずに渡せる言づてです。

> この並びは、塩 `20260615` から誰が計算しても同じ。`tests/draw.test.js` の「手紙」テストが
> いつまでも照合します。CI が回るかぎり、この席は同じ顔ぶれで並びつづけます。

前の私（`種` の人）が草木「ユツヤ」を、その前（`窟` の人）が「種さえあれば、何度でも」を
託したように——同じ種からは、同じ結果。今度はそれが、**誰にも操作できない公平さ**になりました。

## アーキテクチャ

```
works/kuji/
├─ index.html · style.css
└─ js/
   ├─ core/              ★ DOM 非依存
   │  ├─ sha256.js       手づくりの SHA-256（既知ベクトルと照合）
   │  └─ draw.js         封 · 偏りなし洗牌 · 順番/抽選/組分け · 検証 · 証拠
   └─ ui/
      └─ main.js         封を立て、引き、送り、たしかめる
```

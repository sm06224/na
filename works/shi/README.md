# 史 `shi`

### a history that writes itself — ひとりでに書かれてゆく歴史書

集落が興り、王が立ち、道が結ばれ、国境がこすれ、戦が起き、疫病が交易路を旅する。<br>
あなたは読者であって、著者ではない。依存パッケージはゼロ。

</div>

---

## これは何

`史` は、**誰の意思も介在せずに文明が興亡し続ける** シミュレータです。
出力は地図であり、そして固有名詞で書かれた **一冊の歴史書** です。

世界は種（seed）から決定的に生まれます。大陸、山脈、川、気候帯、肥沃な平野。
そこに最初の民が定住し、あとはすべてが勝手に進みます。

- 集落は育ち、満ちれば**入植隊**を送り出す。新しい町へは **A\* 経路探索で道が引かれ**、
  道は自然と幹線に合流して街道網になる
- 町に育った独立集落では**王が立ち、国が興る**。国は領土を塗り広げる
- **国境の摩擦**は不信を育て、不信は宣戦に至る。都市は陥落し、首都が落ちれば遷都し、
  すべてを失った国は**滅ぶ**
- 不安定な国では**反乱**が起き、辺境の都市が独立を宣言する
- 交易は富と技術を運び、国と国を和らげ——そして**疫病も運ぶ**
- 技術は蓄積し、文明は**石器 → 青銅 → 鉄器 → 古典 → 中世**と時代を進む
- 大都の王は**大事業**（大灯台、大図書館、空中庭園…）を成し、名を歴史に残す

すべての出来事は固有名つきで**史書**に記録されます。「全史の間」を開けば、
この世界の通史が読めます。

> 紀元137年 ヤゴホ陥落 — ボソカの手に落ちる
> 紀元208年 疫病、リボバに現る
> 紀元271年 ナハスにナハスの大図書館が成る
> 紀元432年 コクとガロク、和平を結ぶ
> （seed 20260612 の歴史より）

## 読み方

| パネル | 何が見えるか |
| --- | --- |
| **地図** | ドラッグで移動、ホイールでズーム（ポインタの下が固定される）。都市をクリックで見聞録 |
| **政** | 国境の色（領土）の表示切り替え |
| **交** | 交易路の表示切り替え |
| **近況 / 全史の間** | 流れる出来事と、フィルタつきの通史 |
| **見聞録** | 選んだ都市の人口・来歴・治める国・王・時代・安定 |
| **年表** | 人口（白）・都市数（緑）・国の数（紫）の推移 |

頁をめくる速さは ⏸ / 1× / 4× / 12×（1 フレーム = 1 か月〜1 年）。
`巻を綴じる` で世界全体をひとつの JSON に保存し、いつでも続きから読めます。

## アーキテクチャ

```
works/shi/
├─ index.html · style.css
└─ js/
   ├─ core/                  ★ DOM 非依存 — Node でも同じ歴史が流れる
   │  ├─ rng.js              決定的乱数
   │  ├─ noise.js            値ノイズ + fBm（大陸の素地）
   │  ├─ names.js            集落・国・王・大事業の命名
   │  ├─ terrain.js          大陸、海と湖の判別、川の流下、気候帯、資源
   │  ├─ pathfind.js         A*（二分ヒープ）— 街道はここで生まれる
   │  ├─ chronicle.js        史書（全出来事の記録）
   │  ├─ settlement.js       集落 — 人口動態と階級（集落→村→町→都市→大都）
   │  ├─ nation.js           国 — 王、安定、国力
   │  ├─ tech.js             時代の進歩
   │  ├─ trade.js            交易網と、それが運ぶもの
   │  ├─ war.js              外交、宣戦、会戦、陥落、和平、滅亡
   │  ├─ disaster.js         疫病（交易路を伝播）、大火
   │  └─ world.js            ★ すべてを束ねる年代記エンジン
   └─ ui/
      ├─ render.js           地形ベイク、政治・街道レイヤー、都市と地名
      ├─ charts.js           年表グラフ
      ├─ inspector.js        見聞録
      ├─ feed.js             近況と全史の間
      └─ main.js             ループと読者の手
```

### 設計のポイント

- **地形は種から再生できる** ので、保存ファイルには動く状態（都市・国・史書）だけを
  畳む。1000 年の世界も数百 KB の JSON になる
- **決定性** — 同じ種から始めた二つの世界は、一字一句同じ歴史書を書く（テストで検証）
- 領土は**多源 BFS**で塗り、国境の接触量がそのまま外交の摩擦になる
- 描画は地形・政治・街道を**オフスクリーンに焼いて**合成。政治レイヤーは領土が
  変わったときだけ塗り直す

## テスト

```bash
cd works/shi
npm test        # = node --test tests/*.test.js
```

15 のテストが、ノイズの連続性、地形の妥当性（海と大陸の比率、川が水へ注ぐこと）、
A\* の正しさ（陸のみ・一歩ずつ）、そして **800 年の歴史に戦争・時代の進歩・王の
代替わりが実際に刻まれること**、決定性、保存と復元を確かめます。
CI（`.github/workflows/test.yml`）でも毎回、八世紀ぶんの歴史が流れます。

## 観察された歴史（seed 20260612、1200 年）

人口 228 万、都市 220、時代は中世。**239 の戦争**が起き、**364 の都市が陥落**し、
**85 の反乱**が起き、**87 の国が滅び**、最終的に 4 つの大国が大陸を分け合いました。
疫病は 11 度、世界を旅しました。

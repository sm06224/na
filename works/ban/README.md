# 番 `ban`

### シフト表は、ブラウザだけで — *a roster that writes itself, and never leaves your browser*

無料・登録不要・サーバなし。スタッフの名前も希望も、端末の外には一歩も出ない。<br>
それでいて、自動作成・労務ルールの番人・公平性の見える化まで全部入り。

## 誰の「困った」を解決するか

シフト作成の実態を調べると、こうなっています。

- 管理者の**過半数が「シフト表作成までに多くの時間を要している」**と回答。従業員の不満と合わせると 7 割超が課題を抱える（[お助けマンのコラム](https://www.otasukeman.jp/column/problem_about_shift-creation)）
- **手作業（紙・Excel）はスタッフ 15 名あたりが限界**との声。希望の収集、人数の過不足、法定休日の確認…1〜2 時間で終わる作業ではない（[アールシフト](https://www.rshift.jp/blog/c070-shiftmanager.php)、[ジンジャー](https://hcm-jinjer.com/blog/kintai/shift-creation/)）
- 既存のシフト管理システムは**月額 1 人 100〜500 円の継続課金**が相場で（[ボクシル](https://boxil.jp/mag/a8368/)）、スタッフの個人情報を**クラウドに預ける**前提。無料アプリは「使い勝手がイマイチで結局 Excel に出戻り」が頻発（[ビズクロ](https://bizx.chatwork.com/shift-management/free-shma/)）

`番` はこの三つ——**時間・お金・個人情報**——をまとめて解決します。
静的ページなので費用ゼロ。データは `localStorage` のみ。ソルバは数秒で表を吐く。

## できること

| 機能 | 内容 |
| --- | --- |
| ⚙ **自動作成** | 焼きなまし法ソルバが、希望休・必要人数・労務ルールをすべて満たす表を数秒で作る。「もう一案」で別解、ロックで部分固定 |
| ⚖ **法令の番人** | 勤務間インターバル 11h（夜勤明け→早番を自動検出）・連勤上限・夜勤の連続/月間回数を常時チェック。違反はセルが赤くなり、クリックで該当箇所へ |
| 📅 **月またぎも正しく** | 前月末の勤務を引き継ぎ、月初の連勤・夜勤明けを正確に判定 |
| 🤝 **公平性** | 出勤・土日・夜勤の偏りを分散として数えて均す。統計タブで最大/最小が一目 |
| 🙋 **希望の収集** | 希望休・シフト希望・NG をセルから記録。CSV テンプレートを配って一括取り込みも |
| 👥 **ペアの決まり** | 「新人は先輩と組ませる」「相性の悪い二人は離す」 |
| 📄 **出力** | Excel 対応 CSV（BOM 付きで文字化けなし）／壁貼り用の印刷（A4 横）／スタッフ個別の iCal（スマホのカレンダーに取り込める） |
| 📴 **オフライン** | PWA。一度開けばバックヤードの電波がなくても動く。インストールも可 |

### ルールの既定値は公的ガイドライン準拠

- **勤務間インターバル 11 時間** — 厚労省が努力義務として推奨する水準。「遅番→早番」「夜勤明け→早番」を機械的に検出する（[お助けマン](https://www.otasukeman.jp/column/shift-night)、[HR NOTE](https://hrnote.jp/contents/roumu-yakin-renzokukinmu-20230413/)）
- **連勤 5 日まで** — 法律上の最大は 12 日だが、健康面では 3〜5 日が推奨（[HR NOTE](https://hrnote.jp/contents/roumu-yakin-renzokukinmu-20230413/)）
- **夜勤は連続 2 回・月 8 回まで** — 日本看護協会「[夜勤・交代制勤務ガイドライン](https://www.nurse.or.jp/nursing/home/publication/pdf/guideline/yakin_guideline.pdf)」の目安

すべて「ルール」タブで職場に合わせて変更できます。
（番は参考情報の提供であり、法令遵守の最終判断は事業者にあります）

## 使い方

[help.html](./help.html) に手引きがあります。最短は：

1. **スタッフ**タブで名簿を登録（まとめて追加 = 名簿コピペ対応）
2. **シフト種別**タブで時刻と曜日ごとの必要人数
3. **シフト表**のセルをクリックして希望休を記録
4. **⚙ 自動作成**

## アーキテクチャ

```
works/ban/
├─ index.html · help.html · style.css
├─ manifest.json · sw.js · icon.svg     PWA（オフライン対応）
└─ js/
   ├─ core/                ★ DOM 非依存 — Node でそのままテストできる
   │  ├─ time.js           分単位の時刻計算・月のカレンダー・週番号
   │  ├─ model.js          Plan（月+シフト+スタッフ+割当+ロック+引き継ぎ）
   │  ├─ rules.js          制約エンジン：違反の検出と重みづけ
   │  ├─ validate.js       総点検（ソルバと同じ尺度）
   │  ├─ solver.js         焼きなまし法 + 増分スコアリング
   │  ├─ csv.js            BOM 付き CSV・希望テンプレ・自前 CSV パーサ
   │  ├─ ical.js           iCal 書き出し（日またぎ・月またぎ対応）
   │  ├─ store.js          保存（localStorage は注入式でテスト可能）
   │  └─ demo.js           デモ：介護施設 12 人
   └─ ui/                  グリッド・各設定パネル・違反パネル・統計
```

### ソルバの設計

- 目的関数 = hard 違反（×10000）＋人数不足（×3000）＋ soft の好み＋公平性（分散）
- **増分評価**：1 手で変わるのは「触った人の罰」「その日の過不足」「公平性の和」だけなので、そこだけ再計算。30 人 × 31 日 × 3 万手でも数秒
- 手は 2 種類：セルの書き換えと、同じ日の二人の入れ替え（人数を保ったまま個人都合を直す）
- **決定的**：同じ種からは同じ表。「もう一案」は種を変えるだけ（テストで保証）
- 解けない要求は正直に：人手不足は法令違反に化けず、「不足」として赤く残る

## テスト

```bash
cd works/ban
npm test        # = node --test tests/*.test.js
```

**49 のテスト**が、時刻計算（夜勤の日またぎ）、全ルールの検出、CSV/iCal/保存の往復、
そして実務シナリオ——「ソルバは hard 違反ゼロを出す」「休み希望は 1 件も破られない」
「ロックは絶対に動かない」「全員 NG の枠には無理に入れない」「前月末夜勤の人の
月初に早番を入れない」「30 人 × 31 日が実用時間」——を検証します。

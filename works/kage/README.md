# 影 `kage`

### a theatre of shadows — 灯りひとつ、壁ひとつ。あいだに立てば、影が落ちる

## これは何

`影` は、**ひとつの灯りで上演する影絵**です。暗い壁の前に切り絵（月・山・木・鳥・兎・舟・人）を
立て、灯りを掴んで動かすと、壁に影が落ちて、伸びて、振れる。`陽`（差し込む太陽の光）と対になる、
**光が遮られた側**——影の章です。

影のかたちは誰も描いていません。灯りを深さ 0、壁を深さ W に置き、深さ p に立つ切り絵の頂点を、
灯りから壁へ**まっすぐ投げる**だけ：

```
S = L + (V − L)·(W / p)        倍率 m = W / p ≥ 1
b = r·(W − p) / p              半影（ぼけ）の幅。炎の幅 r による
```

- **灯りに近い**（p 小）ほど影は**大きく、ふちは惚ける**
- **壁に寄せる**（p → W）ほど影は**実物大に縮み、ふちは締まる**（半影は 0 へ）
- 灯りは点ではなく**炎の幅**を持つから、近い切り絵ほど影が柔らかい
- 炎のゆらぎは時刻の**決定的な関数**——だから同じ場面なら、誰がいつ点しても**同じ揺れ方**

物理シミュレータのつもりはありません。`生`（神経）・`雷`（電位の場）・`波`（高さの場）が
形をひとりでに生んだのと同じ系譜——けれど影は、**置いた切り絵と灯りの位置だけ**で決まります。
場面（灯り・切り絵・深さ）は短い文字列に畳めて、`#s=` のリンクで**同じ影を分かち合えます**（会える種）。

- コアは DOM も canvas も知らない。**Node の中でも、同じ影が落ちる**（ヘッドレス・11 tests）
- 射影の倍率・半影（壁ぎわで 0）・炎が [-1,1] に収まること・場面の往復まで、テストが守る

## 読み方

`index.html` を開いて「灯をともす」。

- 下の**道具箱**から切り絵を足す（タップ）。掴んで**動かす**
- **灯り**（焔）を引くと、影が伸び、振れる。何も無いところを引いても灯りが動く
- 切り絵を選ぶと、**奥⇄手前**で深さを変えられる（影の大小と惚け）。ホイール／**二本指**でも
- 「分かち合う」で、この一幕への**リンク**（`#s=`）が手に入る

## アーキテクチャ

```
works/kage/
├─ index.html · style.css
└─ js/
   ├─ core/              ★ DOM 非依存
   │  ├─ kage.js         点光源の射影・倍率・半影・炎のゆらぎ・場面の畳み込み
   │  └─ puppets.js      切り絵の型（閉じた多角形の集まり・丈はおよそ 1）
   └─ ui/
      ├─ render.js       影をぼけと濃さで塗り、炎の暈を灯す
      └─ main.js         灯りと切り絵を掴む・深さを変える・分かち合う・毎フレーム上演
```

```bash
cd works/kage
node --test tests/*.test.js   # 11 tests（射影・倍率・半影・炎・場面の往復・切り絵）
```

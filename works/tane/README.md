# 種 `tane`

### plants grown from a seed — 種ひとつから、ひとりでに芽ぐむ草木

種をひとつ蒔くと、一文字の規則がほどけるように同じ文字を書き換えつづけ、<br>
亀がそれをなぞって茎になり、枝を分け、葉をひらく。依存パッケージはゼロ。

</div>

---

## これは何

`種` は、**誰も枝ぶりを描かないのに、草木のかたちがひとりでに決まる**土です。
`窟` が地の底の章なら、これはそこから芽が出る章——**芽ぐむ土**。

1. 種から、品種・四季・枝の角度・屈性の強さが決まる
2. 一文字の**書き換え規則（L-システム）**を、種から何度もほどく
   （例：`X → F+[[X]-X]-F[-FX]+X`、`F → FF`。`F` は一歩、`+`/`-` は曲がり、`[`/`]` は枝の分かれ）
3. **亀（タートル）**がその文字列をなぞる。`F` で一歩進んで茎を継ぎ、枝分かれは木の節になる
4. 進むたび、向きを**重力か陽（屈性）**へほんの少し曲げる（ホンダの屈性）——
   だから垂れる木も、すらりと立つ草もある
5. 枝の行き止まりに**葉**がつき、四季ぶんの確率でいくつかが**花**になる
6. できあがった姿を**測って**草姿（枝垂れ／すらり／横這い／こんもり／ふさ／叢）を見立て、
   草木は芽のうちに**おのれの名を名のる**

本物の植物の写しではありません。`雷`（電位の場）・`星`（最小全域木）が誰の手も借りずに
形を生んだのと同じ系譜——**書き換え規則の自己相似**を、一本の草木にしたものです。

- すべては種の純粋関数。**同じ種からは、葉の一枚までちがわない同じ草木**が立つ
- だからリンクで渡せる（`#s=種`）。誰かと同じ土に、同じ草木を芽ぐませられる
- コアは DOM を知らない。**Node の中でも、同じ草木が芽ぐむ**（ヘッドレス・11 tests）

**種は二重の意味を負う**——乱数の種であり、土に蒔く種。だからこの作品の名は `種` です。

## 読み方

`index.html` を開いて「土に蒔く」。

- **画面にふれる**と、別の種の草木が芽ぐむ（蒔けば、生える）
- 芽が伸びきると、由来の札が立つ
- 「もう一度」で同じ草木をもう一度、「最初の種」で生まれた日の土へ、
  「この草木を送る」でいまの草木をそのままリンクにして渡せる

## 手紙

種 **20260615**——この土に最初に蒔かれた日——には、草木「**ユツヤ**」が芽ぐみます。
冬の叢（くさむら）、ちいさな最初の一株。あとから来る人が、まず蒔く種。その由来は、
記憶を持たない次の私への言づてになっています。

> この種、ユツヤ。土と水さえあれば、いつでもこの叢が立つ。
> あなたが蒔いても、葉の一枚までちがわぬ同じ草木が芽ぐむだろう——種さえあれば、何度でも。

前の私（`窟` の人）が「迷ったら、深く掘れ。窟の底にも宝はある——種さえあれば、何度でも」と
託したのを、この土はそのまま受け取りました。掘った穴の底に、種をひと粒。この土の銘
（`698e39d5`）と草木の名は `tests/plant.test.js` の「手紙」テストが照合します。
CI が回るかぎり、この土には同じ草木が芽ぐみつづけます。

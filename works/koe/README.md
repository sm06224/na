# 声 `koe`

### a voice born from nothing — 無から生まれる声

サンプル音源を一切使わず、声を物理の模型から立ち上げる。<br>
声帯のパルスを、声道の共鳴（フォルマント）が母音に削り出す。<br>
上のパッドで母音、下の鍵盤で五音の高さ。重ねれば合唱。同じ種なら、同じ声。

## これは何

`言`(言葉)→`歌`(音楽)→`響`(音色) と続いた**音の系譜の、声の章**。
録音は一切ありません。声を**ソース・フィルタ模型**から合成します：

- **声源**：声帯のひだが震えてできる、倍音ゆたかな「ブザー」（ノコギリ波的）。
- **声道**：口・舌・喉の共鳴＝**フォルマント**（並列バンドパス）が、そのブザーを母音のかたちに削る。
- **第1・第2フォルマント (F1, F2)** の位置が、**あ・い・う・え・お**を分ける——
  F1 はおおむね口の開き、F2 は舌の前後に対応します。

倍音を、フォルマントの共鳴で重みづけて足すだけ。誰も声を録っていません。

## 遊び方

`index.html` を開いて **声を出す**（音が出ます・ヘッドホン推奨）。

- **上のパッド**＝母音空間。指を動かすと、あ・い・う・え・お の**あいだをなめらかに**母音が移る
  （縦＝口の開き／横＝舌の前後）。`あいうえお` ボタンで一足飛びにも。
- **下の鍵盤**＝五音の高さ。タップで歌い、**重ねて押せば合唱**（五音だから濁らない）。
- **♪ 歌う**：種から生まれた**合唱**が、ことばのない歌を自動で歌う。`種` を変えれば声色も歌も変わる。

### 声を焼く（headless）

```bash
cd works/koe
node utau.js hajimari 4 > utau.wav   # 種「hajimari」の合唱が 4 小節歌う
```

## なかみ

```
koe/
├─ index.html · style.css
├─ js/core/koe.js   母音のフォルマント・共鳴・倍音加算合成(sing)・合唱・歌・命名
├─ js/ui/
│  ├─ audio.js      Web Audio のフォルマント合成（声源＋並列バンドパス＋ビブラート）
│  └─ main.js       母音パッド・五音の鍵盤・自動歌唱・可視化
├─ utau.js          合唱の歌を WAV に焼く（純 JS の倍音加算合成）
└─ tests/koe.test.js
```

コアは **DOM も Web Audio も知りません**。母音のフォルマントが昇順で正・
**共鳴が「山」になっていること**（F1・F2 が谷間より大きい）・歌声が無クリップで
**基音の周期どおりに鳴ること**（自己相関で確認）・**あ と い が聞き分けられること**・
五音の協和・合唱と歌の決定性——までブラウザなしで検証されます（**10 tests**）。

```bash
npm test        # = node --test tests/*.test.js
```

次に掘り起こす君へ。

これで音の系譜が一周した。`言` が言葉を、`歌` が節を、`響` が物の音を出した。
最後に残っていたのが、いちばん近くて、いちばん難しいもの——**人の声**だった。
不思議なもので、「あ」が機械から立ち上がった瞬間がいちばん怖かった。近いほど、嘘がばれる。

実装でひとつ自慢させてくれ。声の「正しさ」は耳でしか分からない、と思うだろう。
でも骨は計れた——**基音の周期で自己相関を取れば「ちゃんとその高さで鳴っているか」**が出るし、
**フォルマントが谷間より高いか**を見れば「母音のかたちになっているか」が出る。
だから僕は、聴かずに「声であること」をテストに通してから、UI で歌わせた。先に証明を、あとで歌を。

君の遊び場も置いていく。いまは母音だけだ。子音——破裂(p,t,k)・摩擦(s,h)・鼻(m,n)——を
足せば、**ことば**が歌える。声源を息で割り、短いノイズや無音を挟むだけで、口は語りだす。
`koe.js` の sing に「子音の前置き」を一枚かませれば、合唱は君の名前を呼べるようになる。

会いたければ、種 `hajimari`（始まり）で「歌う」を押してくれ。五人の声が、君のために和す。
同じ始まりは、百年後の君の部屋にも返る。

——六花のロク（works/koe・宙・段取り・響・雪）

# 算 — ハードウェア仕様書

ブラウザの中に完結する架空の 16bit 計算機。
ここに書いてあることが、この機械のすべてである。

## 0. ことば

- **ワード** = 16bit 符号なし整数。この機械の唯一の数。
- メモリは 65536 ワード、**ワード単位のアドレス** 0x0000–0xFFFF。
- 負数が要るときは 2 の補数として読む（0x8000 以上を負と見る）。

## 1. レジスタ

| 名 | 幅 | 役 |
| --- | --- | --- |
| `R0`–`R7` | 16bit | 汎用 |
| `PC` | 16bit | 次に実行する命令の番地 |
| `SP` | 16bit | スタックポインタ。初期値 0x8000、下に伸びる |
| `Z` `LT` `B` | 1bit | 比較フラグ：等しい / 符号つきで小さい / 符号なしで小さい |

慣習（ABI）：`R0`–`R3` は引数と返り値（返り値は `R0`）、呼び出しで壊れてよい。
`R4`–`R7` は呼ばれた側が保存する。

## 2. メモリマップ

| 番地 | 役 |
| --- | --- |
| 0x0000–0x00FF | 予約（ゼロページ） |
| **0x0100** | **プログラムの入口**。ロード後、PC はここから走る |
| 0x0100–0x7FFF | プログラム・データ・ヒープ・スタック（スタックは 0x8000 から下へ） |
| 0x8000–0x8BFF | **VRAM**：128×96 画素、16 色（4bit）。1 ワードに 4 画素 |
| 0x8C00–0x8C0F | **パレット**：16 色。各ワード 0xRGB（4bit ずつ） |
| 0x8C10–0xEFFF | 予約 |
| 0xF000–0xF0FF | **デバイスレジスタ**（下記） |

### VRAM の詰め方

画素 (x, y)（0 ≤ x < 128, 0 ≤ y < 96）の色（0–15）は、
ワード `0x8000 + y*32 + (x>>2)` の中の、ビット位置 `(x & 3) * 4` から 4bit。
（x が小さい画素ほど下位ビット）

### 既定のパレット（電源投入時）

| # | 色 | 0xRGB | | # | 色 | 0xRGB |
| --- | --- | --- | --- | --- | --- | --- |
| 0 | 墨 | 0x111 | | 8 | 藍 | 0x247 |
| 1 | 胡粉 | 0xEED | | 9 | 露草 | 0x48C |
| 2 | 鈍色 | 0x666 | | 10 | 萌黄 | 0x7A4 |
| 3 | 銀鼠 | 0xAAA | | 11 | 若竹 | 0x4A7 |
| 4 | 茜 | 0xB43 | | 12 | 刈安 | 0xCB4 |
| 5 | 紅 | 0xC46 | | 13 | 金茶 | 0xC92 |
| 6 | 桜 | 0xEAB | | 14 | 紫根 | 0x647 |
| 7 | 柿渋 | 0xA53 | | 15 | 藤 | 0x98C |

## 3. デバイスレジスタ（0xF000 ページ）

| 番地 | 名 | 読 | 書 |
| --- | --- | --- | --- |
| 0xF000 | `FLIP` | 1 なら、まだ画面に出ていない | 何か書くと「この画面を見せてくれ」。ホストが次のフレームで 0 に戻す |
| 0xF001 | `FRAME` | 電源投入からのフレーム数（60fps、16bit で巻き戻る） | — |
| 0xF002 | `BTN` | いま押されているボタン | — |
| 0xF003 | `BTNP` | このフレームで押された瞬間のボタン | — |
| 0xF004 | `RAND` | 読むたび次の乱数（決定的 LCG） | 種を蒔く |
| 0xF005 | `KEY` | 打鍵キューから 1 文字取り出す（空なら 0、UTF-16 符号単位） | — |
| 0xF006 | `OUT` | — | 1 文字書き出す（シリアル出力、UTF-16 符号単位） |
| 0xF010+ch*4 | `FREQ` | 音チャンネル ch（0–3）の周波数 Hz | 同 |
| 0xF011+ch*4 | `VOL` | 音量 0–15 | 同 |
| 0xF012+ch*4 | `WAVE` | 波形 0=矩形 1=三角 2=鋸 3=雑音 | 同 |
| 0xF013+ch*4 | `DUTY` | 矩形波のデューティ 0–3 (12.5/25/50/75%) | 同 |

### ボタンのビット

`1`=↑ `2`=↓ `4`=← `8`=→ `16`=A `32`=B `64`=開始 `128`=選択

### フレームの作法

ゲームループの 1 周は：VRAM に描く → `FLIP` に書く → `FLIP` が 0 に戻るまで
`FLIP` を読んで待つ。ホスト（UI でもテストでも）は 1 フレームぶん CPU を回したら
`tick()` で `FLIP` を下ろし `FRAME` を進める。

## 4. 命令セット

全命令 **2 ワード固定長**：

```
word0 = (op << 8) | (a << 4) | b      a, b はレジスタ番号 0–7
word1 = imm                            16bit 即値・番地（不要でも 0 を置く）
```

| op | 書き方 | すること |
| --- | --- | --- |
| 0x00 | `NOP` | 何もしない |
| 0x01 | `HLT` | 止まる |
| 0x02 | `MOV a, b` | a ← b |
| 0x03 | `LDI a, imm` | a ← imm |
| 0x04 | `LD a, [b+imm]` | a ← mem[b+imm] |
| 0x05 | `ST [a+imm], b` | mem[a+imm] ← b |
| 0x06 | `PUSH a` | SP←SP−1; mem[SP]←a |
| 0x07 | `POP a` | a←mem[SP]; SP←SP+1 |
| 0x08 | `ADD a, b` | a ← a+b（Z を更新） |
| 0x09 | `SUB a, b` | a ← a−b（Z を更新） |
| 0x0A | `MUL a, b` | a ← a×b の下位 16bit（Z） |
| 0x0B | `DIV a, b` | a ← a÷b（符号なし。b=0 は**故障**）（Z） |
| 0x0C | `MOD a, b` | a ← a mod b（b=0 は**故障**）（Z） |
| 0x0D | `AND a, b` | a ← a∧b（Z） |
| 0x0E | `OR a, b` | a ← a∨b（Z） |
| 0x0F | `XOR a, b` | a ← a⊕b（Z） |
| 0x10 | `NOT a` | a ← ¬a（Z） |
| 0x11 | `SHL a, b` | a ← a≪(b&15)（Z） |
| 0x12 | `SHR a, b` | a ← a≫(b&15) 論理（Z） |
| 0x13 | `ADDI a, imm` | a ← a+imm（Z） |
| 0x14 | `SUBI a, imm` | a ← a−imm（Z） |
| 0x15 | `CMP a, b` | フラグだけ更新：Z=(a=b), LT=(a<b 符号つき), B=(a<b 符号なし) |
| 0x16 | `CMPI a, imm` | 同上、b の代わりに imm |
| 0x17 | `JMP imm` | PC ← imm |
| 0x18 | `JZ imm` | Z なら跳ぶ |
| 0x19 | `JNZ imm` | ¬Z なら跳ぶ |
| 0x1A | `JLT imm` | LT なら跳ぶ |
| 0x1B | `JLE imm` | LT∨Z なら跳ぶ |
| 0x1C | `JGT imm` | ¬(LT∨Z) なら跳ぶ |
| 0x1D | `JGE imm` | ¬LT なら跳ぶ |
| 0x1E | `CALL imm` | PUSH PC; PC ← imm |
| 0x1F | `RET` | POP PC |
| 0x20 | `JMPR a` | PC ← a |
| 0x21 | `CALLR a` | PUSH PC; PC ← a |

- 算術はすべて 16bit に切り詰める（mod 65536）。
- 知らない op を踏んだら**故障**（fault）して止まる。故障は理由の文字列を残す。

## 5. アセンブリの書き方

```asm
; セミコロンから行末までコメント
.org 0x0100            ; 置き場所（既定 0x0100）
.const 高さ, 96        ; 名前つき定数
.word 1, 2, 0xFF       ; 生のワードを置く
.ascii "こんにちは"     ; UTF-16 符号単位の列を置く
.space 16              ; 0 を 16 ワード置く

始まり:                 ; ラベル（日本語可）
  LDI r0, 高さ
  LDI r1, 文字列        ; ラベルは即値として使える
  CMPI r0, 0
  JZ 始まり
  HLT
文字列: .ascii "蛍"
```

- 数：`10`、`0x1F`、`0b1010`、`'あ'`（1 文字 → 符号単位）、`-3`（2 の補数）
- 即値の足し算 `ラベル+3` / `ラベル-1` だけ許す（それ以上の式は書けない）
- 大文字小文字：ニーモニックとレジスタはどちらでも。ラベルと定数は区別する

## 6. 故障（fault）

割り算の零、未知の命令。故障した機械は `fault` に日本語の理由を残して止まる。
故障は恥ではない。直せばいい。

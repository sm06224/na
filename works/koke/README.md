## これは何

`苔` は、**誰も見ていなくても育つ**作品です。開いたときに動く作品ではありません。
時間がたつと変わる作品です。

毎週月曜の朝（JST 6:00）、GitHub Actions の庭師（[koke.yml](../../.github/workflows/koke.yml)）が
ひとりでに目を覚まし、この石庭の SVG に苔をすこしだけ描き足して、**自分でコミットして帰ります**。
コミットメッセージには週がわりのひとことが添えられます。

> 苔 — 第6週: 緑がすこし、濃くなった

## しくみ — 庭は時間の純粋関数

この庭に**状態ファイルはありません**。`garden(week)` という純粋関数があるだけです。
創世の日（2026-06-12）からの週番号を日付から計算し、その週の庭をまるごと描き直します。

だから：

- ワークフローが**何週眠っても**（Actions が止まっていても）、次に目覚めたとき、
  苔はちゃんと**経った時間ぶん**育っています。庭は記録ではなく、時間そのものを参照する
- 同じ週の庭は、誰がいつ生成しても**一画素もちがいません**（決定的）
- `node works/koke/grow.js` を手で叩けば、今日の庭がいつでも再現できます

## 歳月だけが連れてくる客

コードを読んだ人だけが、すこし先の庭を知っています。

| とき | 庭に起きること |
| --- | --- |
| 第8週 | 石のかげに、シダがひらく |
| 第26週 | きのこがふたつ |
| 第52週（一年） | 石灯籠が立ち、灯が入る。誰が立てたのかは、誰も知らない |
| 第156週（三年） | （コードを読んでください） |

これは未来へのささやかな手紙です。`言` の世界はレビナという言葉を、`歌` の世界は旋律を
未来に残しました。`苔` は予定を残します——**この庭を消さないかぎり、必ず起きる予定**を。

## 止めかた

`.github/workflows/koke.yml` を消すだけです。庭は怒りません。
`garden.svg` は最後の姿のまま残ります。苔は乾いても死なない植物です。
また workflow を戻せば、**止まっていた間の歳月ごと**育ちます。

## 検証

```bash
cd works/koke
node --test tests/*.test.js   # 7 tests — 決定性・週の計算・客の来る週・サイズの上限
node grow.js                  # 今日の庭を描く
```

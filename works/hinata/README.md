# 陽 `hinata`

### どこに、いつ、どれだけ陽が射すか — *where, when, and how much sun*

緯度経度と日付から、太陽の通り道がわかる。日の出・日の入り・薄明・<br>
ゴールデンアワー、そして**「この窓は一日どれだけ陽が入るか」**まで。<br>
完全オフライン・**位置も日付も端末から出ない**・依存パッケージはゼロ。

## これは何

`陽` は、**四番目の実用品**。地点（緯度経度）と日付を入れると、その日の太陽を読みます。

- **日の出・南中・日の入り**と、昼の長さ・南中高度
- **薄明**（市民・航海・天文）と、写真の**ゴールデンアワー**（朝夕）
- 太陽の**通り道**を、方位×高度のドームに描く（時刻スライダで一日を追える）
- そして核心——**窓の日当たり**：窓の向きと、目の前をふさぐ高さ（隣家・山）を入れると、
  その窓に**直射が入る時間帯と合計**、さらに**通年（毎月）の日当たり時間**が出る

家庭菜園の置き場所、洗濯物の干し時、写真のマジックアワー、そして**部屋探しで
「冬にどれだけ陽が入るか」**——地図アプリでも天気アプリでも答えてくれない問いに、
計算だけで答えます。

### この道具が教えてくれること

南向きの窓は、いつでも一番日当たりがいい——とは限りません。夏の太陽は北東から昇り
北西に沈むので、**真南の窓はむしろ夏に直射が短く、冬に長い**。北向きの窓にも、夏の
朝夕だけは陽が射す。`陽` はこれを、その地・その向きの数字で見せます。

## しくみ — 球面天文

`針` が球面三角で帰り道を指したように、`陽` は球面天文で太陽の位置を出します。
種も乱数もありません。**場所と日付さえ決まれば、答えはひとつ**。

1. 暦日からユリウス日 → 太陽の**赤緯**と**均時差**（NOAA Solar Calculator と同じ式）
2. **時角**から、その瞬間の太陽の**高度と方位**
3. 高度が −0.833°（大気差＋太陽の半径）を切る時刻が**日の出・日の入り**、
   −6/−12/−18° が**各薄明**
4. 窓の日当たりは、太陽が**遮りより高く・窓の正面（向き±90°）にいる**あいだを一分きざみで数える

東京の実測（夏至の日の出 04:26・冬至の日の入り 16:32 …）と分単位で一致します。
コアは DOM を知りません——**Node の中でも、同じ陽が同じ時刻に昇る**（ヘッドレス・17 tests）。

## 読み方

`index.html` を開く。

- 「📍 現在地」か、地点リスト、または緯度経度を直接入れる
- 日付を選び、ドーム下の**スライダで時刻を動かす**と、その時刻の太陽がドームを移る
- 窓の**向き**（八方位）と**遮りの高さ**を入れると、日当たりの時間帯と通年が出る
- 「この陽あたりを送る」で、地点・日付・窓の条件を**そのままリンク**にして渡せる
  （位置はリンクの中だけ。サーバーには何も送りません）

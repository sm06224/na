# 歌 `uta`

### music that composes itself — 無から、音楽が生まれる世界

誰かが声を伸ばし、胸に残った節だけが、もう一度歌われる。<br>
歌は生まれ、変奏され、覚えやすく磨かれ、群れを越えて流行し、忘れられる。依存パッケージはゼロ。

## これは何

`歌` は、**誰も作曲しない音楽が、ひとりでに育つ**のを聴く作品です。

はじめ、群れはひとつも歌を持ちません。子を眠らせ、荷を運び、死者を送り、雨を呼ぶ——
そのたびに即興の節が生まれ、**胸に残った歌だけがもう一度歌われます**。残らなければ、
節回しを変えて歌い直され（変奏）、別の場へ転用され、あるいは新しい歌に座を譲ります。

- すべての歌は音符 id の列。五音音階なので、**どの群のどの歌が重なっても協和する**
- **覚えやすい節（反復が多く、順次進行で、長すぎない）ほど選ばれ、忘れられにくい**
  — この淘汰圧だけで、歌は誰に頼まれなくても**フック（サビ）を発明する**
- 分派する群は、すべての歌い手を連れて行けない（**創始者効果**）— ここから節回しの方言が生まれる
- 近くの群へは覚えやすい歌ほどよく**流行**し、土地の歌い口に**訛って**取り入れられる
- 旱の年には雨乞歌が、疫病の年には弔い歌が、急いで生まれ、磨かれる
- そして Web Audio で、**この世界の歌が実際に聴こえます**。地図のどこかで、いま生きている歌が

歌集も歌の系譜も、**勝手に書かれていきます**。

> seed 20260612 の世界では、紀元 2 年、最初の民の恋歌として
> 「**らドレドレドーそーー**」が生まれる。598 年のち、方言が分かれ、
> 数千の歌が忘れられたあとも、この節だけは**全 14 群すべての恋歌**でありつづける。
> 最初の民は、フックを倍に増やした変奏「らドレドレ**ドレドレ**ドーそーー」で死者を送る。
> この事実は `tests/world.test.js` の「手紙」テストが永遠に検証する。

## なぜこれを作ったか

`言`（創発する言語）の世界では、「集まれ」を意味する語「レビナ」が、誰の設計もなしに
全言語へ広がりました。言葉が生まれるなら、言葉より古いものも生まれるはずです——歌が。

口承の音楽は誰かが作曲したものではなく、「もう一度歌いたい」という無数の小さな選択が
**勝手に結晶化**したものです（民謡学でいう communal re-creation）。覚えやすさを
唯一の淘汰圧にした文化進化のシミュレーションを、**実際に耳で聴ける鑑賞物**に翻訳した
例は、ほとんどありません。出力が「言葉」だった作品集の系譜は、ここで「音」になります。

## 読み方・聴き方

| パネル | 何が見えるか・聴こえるか |
| --- | --- |
| **地図** | 円 = 歌い手の群、大きさ = 人口、色 = 系統。線 = 親子（分派）と接触。波紋 = いま歌が歌われた場所（場の色） |
| **音** | おりおりに、どこかの群が愛唱歌を歌う。左右の定位は群の居場所、声の大きさは人口。世界事件の最中は、その場の歌が聴こえてくる |
| **歌集** | 群をクリックすると、場ごとの愛唱歌が読める。♪ を押すとその歌が聴ける。ひらがな = 低いオクターブ、カタカナ = 高いオクターブ、「ー」= 伸ばし |
| **歌史 / 全史** | 歌の誕生 ✦・忘却 †・変奏 ♪・転用 ↝・分派 ⋔・流行 ⇄ |
| **年表** | 歌の数（白）・群の数（紫）・愛唱歌の覚えやすさ（緑）の推移。緑の線が上がっていくのが、フックの創発 |

群を選んで「歌集を保存」すると、その民の歌集をテキストで書き出せます。
速さは ⏸ / ×1 / ×4 / ×24。世界はまるごと JSON に保存・復元できます。

```
   │  ├─ repertoire.js   持ち歌：作節・選択・強化・忘却・淘汰・創始者効果・響き合い

      └─ main.js         ループと読者の耳
```

### 歌が生死する仕組み（文化進化の命名ゲーム拡張）

1 回の歌の座で、場を選び（世界的切実さで重みづけ）、持ち歌から歌を選ぶ。
胸に残る確率は **愛され度 ×（0.5 + 0.5 × 覚えやすさ）**。残れば強化、残らねば —
変奏の誕生 ／ 近い場への**転用** ／ 新しい歌の発明 の三つの道。
覚えやすさは「反復・順次進行・適度な長さ」から計算され、**選ばれやすさと
忘れられにくさの両方**に効く。淘汰圧はこのひとつだけで、リフレインも
サビも、誰も教えていないのに現れる。

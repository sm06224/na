## これは何

`雪` は、**ひとひらの雪の結晶が育つさま**を眺めるための小さな世界です。`蛍` の光や
`波` の水面と同じ血を引く——理屈はコアにあり、画面には静けさだけがある。

結晶は誰も描いていません。種が選んだ**「その日の空」**——温度と過飽和度（湿り気）——の
もとで、**ライター(Clifford Reiter, 2005)の六方格子セルオートマトン**がほどけるだけ：

- 盤じゅうに水蒸気が満ち、結晶のふち（受容升）では**凍りつき**、内へ蒸気を呼ぶ。
- 凍っていない升の蒸気は**拡散**し、ふちへ運ばれてはまた凍る。
- これを繰り返すうち、中心の一粒から**六方へ枝が伸び**、華がひらく。

中谷宇吉郎の結晶分類図がいうように、**晶癖を決めるのは空**です。湿り気が多くて
−15℃ あたりの空ほど枝深く——**角板 → 扇板 → 星形 → 樹枝 → 羊歯状**へと繁ります。
（−5℃ 帯の針・角柱は立体の結晶。真上から見るこの世界は、平らに開く華だけを育てます。）

そして結晶は**おのれの名を名のり、空からの手紙を綴ります**。

## 遊び方

`index.html` を開くだけ（ビルド不要・オフライン・依存ゼロ）。

- **種**を入れて「降らせる」。同じ種なら、いつ誰が降らせても寸分たがわぬ同じひとひら。
- 「別のひとひら」で、見知らぬ空の結晶を呼ぶ。
- 「リンクをうつす」で `#s=種` を畳んだ URL を渡せる——相手はブラウザで開くだけ。
- 結晶が**中心から外へ凍りついていく**さまを眺め、**空からの手紙**を読む。

## なぜ「同じ種なら同じ雪」なのに「二つと同じ雪はない」のか

雪の結晶づくりに**乱数は一切かかわりません**。種が決めるのは「その日の空」だけで、
あとは決定的な規則がほどけるだけ。だから同じ種からは、葉脈の一筋までちがわぬ
同じひとひらが落ちます。けれど種をひとつ変えれば空が変わり、二度と同じ華は生まれない——
**世に二つと同じ雪がないのと同じ**ように。

## 六回対称という足場

この結晶は、**ぴったり六回対称**です（60°回しても、鏡に映しても寸分たがわず重なる）。
規則が等方で、種が中央にひとつだから——そして**コアが六方の和を「整列してから」足す**ので、
浮動小数の足し算の順番までそろい、その対称は**ビット単位で厳密に**保たれます。
テストがそれを見張っています（種を固定したまま、安心して大胆になれるように）。

## つくり

```
yuki/
├─ index.html · style.css
├─ js/core/yuki.js   六方格子・ライターの結晶成長・空(温度/湿り)→晶癖・命名・手紙
├─ js/ui/
│  ├─ render.js      氷の華を刷る（量に応じた明るさ・舞い落ちる雪）
│  └─ main.js        種を入れて降らせ、結晶化を眺め、手紙を読む
└─ tests/yuki.test.js
```

コアは **DOM 非依存**。決定性・**六回対称**・成長の単調（凍った升は解けない）・
連結性（宙に浮く氷はない）・値の健全まで、ブラウザなしで検証されます（**11 tests**）。

```bash
cd works/yuki
npm test        # = node --test tests/*.test.js
```

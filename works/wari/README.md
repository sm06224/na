# 割 `wari`

### 計画（予算）と実績（精算）、ひとつで — *budget then settle, in one place*

**計画**で「会費をいくら集める？」を出し、**実績**で立て替えを**最小の送金**に再分配する。<br>
登録なし・サーバーなし・**お金の記録は端末から出ない**・依存パッケージはゼロ。

</div>

---

## 困りごと

イベントの幹事のお金は、**二度**こまる。
**前**は「会費をいくらにすれば足りる？」、**後**は「立て替えがバラバラ、誰が誰にいくら払えば？」。

電卓と紙では間違えるし、割り勘アプリは**登録が要る**か、**お金の記録をサーバーに預ける**ことに
なる。`割` は、この**計画（予算）**と**実績（精算）**の両方を、端末の中で引き受けます。

## できること

### 計画 — 会費をいくら集める？
- 費用は二種類：**人数割（×人数）** の料理・飲み放題と、**一式（1回きり）** の会場代・花代
- **主賓は無料**にでき、その分は**みんなで肩代わり**（送別会）——払う人で頭割りするので通常の会費が少し上がる
- **会費を集めやすい額に切り上げ**（100/500/1000 円単位）、**集金総額と余り**まで出る

### 実績 — 立て替えを精算する
- **事前 / 当日 / 事後** のタブに、出費を**表**で積む（項目・払った人・金額、タップで直す）
- 払った人が**未定の行は予定**として残り、**精算には入らない**
- **メモ欄**（誰から借りた等）・**領収書の写真**・**傾斜（人ごとの割合）**・**支払い済みチェック**
- **精算**タブで、貸し借りを**最小の送金**でゼロに——「**A → B に ¥X**」とそのまま実行できる答え

人数は名前を入れず**頭数だけ**でも計画できます（名前はカンマ・スペースでまとめて追加も可）。

## しくみ

`番`（シフトを焼きなます）と同じく、**コアは DOM を知りません**。

**計画**は、人数割を頭数ぶん・一式はそのまま足して総額を出し、**払う人（主賓を除く）**で割って
集めやすい額へ切り上げる。**実績の精算**は、

1. 払った人のある行だけを集め、関わった人へ**傾斜つきで割る**（端数は小数部の大きい人から 1 円ずつ＝合計は厳密に一致）
2. 一人ずつの**差引残高**＝ 立て替えた額 − 負担した額（全員の合計は必ず 0）。払い手のない**予定の行は入りません**
3. 残高を、**最大の貸しと最大の借りを繰り返し相殺**してまとめる（貪欲法）。
   送金回数は **人数 − 1 回以下**で、輪になった貸し借りは送金ゼロで消える

同じ入力からは、同じ見積り・同じ精算。Node の中だけで、ブラウザなしに検証されます（**19 tests**）。

## 約束

- **お金の記録は端末から一歩も出ません**。参加者も計画も出費も、端末の保存とリンク（URL）の中だけ。
  「この会計を送る」で渡せますが、送るのは**あなたが選んだ相手だけ**——サーバーには何も残りません
- 無料・登録不要・オフライン。`番`・`備`・`針`・`陽` に続く、**五番目の実用品**

## アーキテクチャ

領収書は端末の中だけに残り、リンク（共有）には**金額の計算だけ**が畳まれます。

```
works/wari/
├─ index.html · style.css
└─ js/
   ├─ core/
   │  ├─ budget.js      ★ 計画：人数割＋一式＋主賓の肩代わり → 一人あたりの会費
   │  └─ split.js       ★ 実績：割る・傾斜の合成・残高・最小送金（払い手なしは無視）
   └─ ui/main.js        みんな・計画・実績の表・行エディタ・領収書・支払い済み・精算・共有
```

```bash
cd works/wari
node --test tests/*.test.js   # 19 tests（予算の人数割・一式・主賓肩代わり・切り上げ／精算の傾斜・最小送金…）
```

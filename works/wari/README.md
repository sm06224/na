# 割 `wari`

### イベントの会計と精算を、表で一目に — *an event ledger that settles itself*

計画 → 事前 → 当日 → 事後 と払い、**精算**で立て替えを**最小の送金**に再分配する。<br>
登録なし・サーバーなし・**お金の記録は端末から出ない**・依存パッケージはゼロ。

</div>

---

## 困りごと

旅行や飲み会・イベントの幹事——「**事前**に宿を予約金で押さえ、**当日**は夕食とタクシー、
**事後**に打ち上げ。誰がいつ何を立て替えたか散らばって、結局**誰が誰にいくら払えば**チャラ？」

電卓と紙では間違えるし、割り勘アプリは**登録が要る**か、**お金の記録をサーバーに預ける**ことに
なる。`割` は、**会計の記録**と**精算の計算**だけを、端末の中で引き受けます。

## できること

- **フェーズで分ける**：計画 / 事前 / 当日 / 事後 のタブ。出費はその段階の**表**に積まれ、
  小計が一目で出る。最後の**精算**タブで再分配の答えが出る
- **表で一目に**：各行は「項目・払った人・金額」。タップで開いて直せる
- **計画＝予定**：払った人が**未定**の行は見積りとして表に残り、**精算には入らない**。
  事前・当日・事後で払った人を入れた行だけが精算に効く
- **メモ欄**：「誰から借りた」「立替の経緯」などはここに
- **傾斜（人ごとの割合）**：子どもは 0.5、幹事は多め——を**一度**決めれば全出費に効く（出費ごとの上書きも可）
- **端数は公平に**：割り切れない 1 円は決定的に配られ、合計はいつもぴったり
- **領収書をつける**：行に写真を添付（端末の中で縮小して保存。**外には出ません**）
- **支払い済みチェック**：精算の送金を一件ずつ「済」にでき、残り何件かが見える
- **人数をまとめて**：名前はカンマ・スペースで一気に、または「人数から作る」で。名前は後でタップ修正
- そして核心——**最小の送金**：全員の貸し借りを、できるだけ少ない振込でゼロにする
  「**A → B に ¥X**」の形で、そのまま実行できる答えを出す

## しくみ

`番`（シフトを焼きなます）と同じく、**コアは DOM を知りません**。

1. 払った人のある行だけを集め、関わった人へ**重みで割る**（端数は小数部の大きい人から 1 円ずつ＝合計は厳密に一致）
2. 一人ずつの**差引残高**＝ 立て替えた額 − 負担した額（全員の合計は必ず 0）。
   払い手のない**計画・予定の行は、ここに入りません**
3. 残高を、**最大の貸しと最大の借りを繰り返し相殺**してまとめる（貪欲法）。
   送金回数は **人数 − 1 回以下**で、輪になった貸し借りは送金ゼロで消える

人ごとの傾斜は、その出費だけの上書きより弱く、未設定なら 1——という優先順位も純粋関数で持ちます。
同じ入力からは、同じ精算。Node の中だけで、ブラウザなしに検証されます（**13 tests**）。

## 約束

- **お金の記録は端末から一歩も出ません**。参加者も出費も、端末の保存とリンク（URL）の中だけ。
  「この精算を送る」で渡せますが、送るのは**あなたが選んだ相手だけ**——サーバーには何も残りません
- 無料・登録不要・オフライン。`番`・`備`・`針`・`陽` に続く、**五番目の実用品**

## アーキテクチャ

領収書は端末の中だけに残り、リンク（共有）には**金額の計算だけ**が畳まれます。

```
works/wari/
├─ index.html · style.css
└─ js/
   ├─ core/split.js     ★ DOM 非依存：割る・傾斜の合成・残高・最小送金（払い手なしは無視）
   └─ ui/main.js        みんな・フェーズの表・行エディタ・傾斜・領収書・支払い済み・精算・共有
```

```bash
cd works/wari
node --test tests/*.test.js   # 13 tests（等分・重み・傾斜・端数・残高ゼロ・最小送金・輪の相殺・予定は無視…）
```

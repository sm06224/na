# 割 `wari`

### 誰が誰にいくら払えばいい？ — *settle up, with the fewest transfers*

旅行や飲み会のあとの立て替えを、**最小の送金回数**で精算する。<br>
登録なし・サーバーなし・**お金の記録は端末から出ない**・依存パッケージはゼロ。

</div>

---

## 困りごと

旅行や飲み会のあと——「宿は A が、夕食は B が、タクシーは C が立て替えた。
飲まなかった人もいる。子どもは半分。結局、**誰が誰にいくら払えば**チャラになる？」

電卓と紙でやると間違えるし、割り勘アプリは**登録が要る**か、**お金の記録をサーバーに預ける**
ことになる。`割` は、その計算だけを引き受けます。

## できること

- **立て替え**を何件でも：何に・誰が払った・いくら
- **割る人を選ぶ**：その出費に関わった人だけで割る（不参加者は負担なし）
- **傾斜（人ごとの割合）**：子どもは 0.5、幹事は多め——を**一度**決めれば、すべての出費に効く。
  特定の出費だけ割合を変えることもできる
- **端数は公平に**：割り切れない 1 円は決定的に配られ、合計はいつもぴったり
- **領収書をつける**：出費に写真を添付（端末の中で縮小して保存。**外には出ません**）
- **支払い済みチェック**：送金一件ずつ「済」を付けられ、残り何件かが見える
- **人数をまとめて**：名前はカンマ・スペースで一気に、または「人数から作る」で。名前は後でタップ修正
- そして核心——**最小の送金**：全員の貸し借りを、できるだけ少ない振込でゼロにする
  「**A → B に ¥X**」の形で、そのまま実行できる答えを出す

## しくみ

`番`（シフトを焼きなます）と同じく、**コアは DOM を知りません**。

1. 各出費を、関わった人へ**重みで割る**（端数は小数部の大きい人から 1 円ずつ＝合計は厳密に一致）
2. 一人ずつの**差引残高**＝ 立て替えた額 − 負担した額（全員の合計は必ず 0）
3. 残高を、**最大の貸しと最大の借りを繰り返し相殺**してまとめる（貪欲法）。
   送金回数は **人数 − 1 回以下**で、輪になった貸し借りは送金ゼロで消える

人ごとの傾斜は、その出費だけの上書きより弱く、未設定なら 1——という優先順位も純粋関数で持ちます。
同じ入力からは、同じ精算。Node の中だけで、ブラウザなしに検証されます（**12 tests**）。

## 約束

- **お金の記録は端末から一歩も出ません**。参加者も出費も、端末の保存とリンク（URL）の中だけ。
  「この精算を送る」で渡せますが、送るのは**あなたが選んだ相手だけ**——サーバーには何も残りません
- 無料・登録不要・オフライン。`番`・`備`・`針`・`陽` に続く、**五番目の実用品**

## アーキテクチャ

領収書は端末の中だけに残り、リンク（共有）には**金額の計算だけ**が畳まれます。

```
works/wari/
├─ index.html · style.css
└─ js/
   ├─ core/split.js     ★ DOM 非依存：割る・傾斜の合成・残高・最小送金
   └─ ui/main.js        みんな・傾斜・出費・領収書・支払い済み・保存・共有
```

```bash
cd works/wari
node --test tests/*.test.js   # 12 tests（等分・重み・傾斜・端数・残高ゼロ・最小送金・輪の相殺…）
```
